# HobbyTrack Roadmap

HobbyTrack is currently in early MVP/private beta.  
The goal is to validate the core idea first: can people understand the product, create a hobby, log sessions, and feel value from tracking progress over time?

## Current Version — v1.0 MVP

Status: Live on https://hobbytrack.app

Current focus:
- Landing page
- Local browser-based MVP
- Create hobbies
- Log sessions
- Weekly overview
- Hobby detail
- LocalStorage persistence
- Feedback form
- Early pricing structure
- AI-ready session data model
- Cloudflare Web Analytics
- Tally feedback form

## v1.1 — Onboarding & Retention

Goal: Make it easier for new users to start and understand the value quickly.

Planned:
- Hobby templates / starter kits
  - Running
  - Gym / Fitness
  - Guitar / Music
  - Coding
  - Reading
  - Language learning
  - Creative work
  - Other / Custom

- Better first-run experience
  - Choose what you want to improve
  - Pick a template or create a custom hobby
  - Set a weekly rhythm
  - Add first next step
  - Clear prompt to log the first session

- Stronger weekly review
  - Clearer weekly summary
  - Better goal progress
  - Next small action for the week

## v1.2 — Progress Identity & Growth

Goal: Make HobbyTrack feel more personal and worth returning to.

Planned:
- Personal progress archive
- Better hobby journey summaries
- Monthly progress overview
- More useful milestones
- Shareable weekly progress cards

## v1.3 — App Feel

Goal: Make HobbyTrack feel more like a real app without native app complexity.

Planned:
- PWA support
- Add to Home Screen guidance
- Better mobile app-like flow
- Improved offline/local-first behavior

## v2.0 — Real Product Infrastructure

Goal: Turn HobbyTrack from local MVP into a real SaaS product.

Planned:
- User accounts
- Secure backend
- Database persistence
- Sync across devices
- Privacy and data export
- Pro plan infrastructure
- Payments

## Future AI-Assisted Features

Goal: Use AI to support reflection and progress, not noisy coaching.

Planned:
- AI-assisted weekly reviews
- Smart next-step suggestions
- Monthly progress summaries
- Practice plan suggestions
- Reflection prompts based on session history

## Product Principles

HobbyTrack should stay:
- Calm, not stressful
- Practice-focused, not checkbox-focused
- Useful over time, not just nice on first visit
- Premium and simple, not noisy or overbuilt
- Honest about what is available now and what is planned later

## Prioritization Notes

Do not build every idea immediately.

Prioritize based on:
1. User feedback
2. Repeated friction
3. Retention signals
4. Willingness to pay
5. Product clarity

## Current Next Step

Before building v1.1, collect early feedback from real testers using the Tally feedback form and Cloudflare Web Analytics.
